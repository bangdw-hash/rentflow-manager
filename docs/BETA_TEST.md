# rentflow V2 — 전문가 베타테스트 & 기능 결정 보고서

> 목적: V1(매물·임차인·계약·관리비·알림·보고서) 출시 후, 5개 전문가 관점으로 베타테스트를
> 진행하고 핵심 인사이트를 도출 → 경쟁 서비스 벤치마킹 → V2 돌파구 기능을 결정·개발한다.

---

## 1. 전문가 패널 & 핵심 피드백

### 🎨 전문 디자이너
- **문제**: 기본 시스템 서체 + 제네릭 Tailwind 룩으로 "촌스럽다". 위계/여백/숫자 정렬이 약함.
- **요청**: 한국형 모던 서체, 타이트한 자간, 일관된 카드/버튼 토큰, 금액의 등폭 정렬.
- **핵심 인사이트**: *신뢰감은 타이포그래피에서 시작한다.* 금융성 데이터는 정렬·자간이 곧 완성도.

### 🏠 부동산 중개인
- **문제**: "지금 당장 뭘 해야 하는지"가 흩어져 있음(만료/공실/미납이 각 페이지에 분산).
- **요청**: 만료 파이프라인 + 공실 회전(turnover)을 한 화면에서 행동까지 연결.
- **핵심 인사이트**: *관리는 목록이 아니라 "다음 행동"이 핵심이다.*

### 📈 상품 마케터
- **문제**: V1엔 "와우 포인트"(차별적 가치)가 약함 — 경쟁앱과 기능이 평이하게 겹침.
- **요청**: 재방문을 만드는 루프(이번 달 할 일) + 한 줄 가치제안.
- **핵심 인사이트**: *리텐션은 "이번 달 할 일" 같은 반복 행동 루프에서 나온다.*

### 💼 전문 임대인
- **문제**: 현금흐름/수익률이 보이지 않음. 매물별로 얼마를 버는지, 점유율이 어떤지 즉시 모름.
- **요청**: 월/연 예상수입, 점유율, 매물별 수익률 한눈에.
- **핵심 인사이트**: *임대인은 "수익률과 현금흐름"으로 의사결정한다.*

### 🧾 세무 관점 (세무사)
- **문제**: 5월 종합소득세 시즌에 연간 임대소득을 매번 수기로 집계. 경쟁앱도 약함.
- **요청**: 연도별 월별 임대수입 집계 + 엑셀 내보내기 + 간주임대료 안내.
- **핵심 인사이트**: *세무 신고용 연간 리포트는 "지금 시장에 비어 있는" 강력한 차별점.*

## 2. 경쟁 서비스 벤치마킹

| 서비스 | 강점 | rentflow가 취할 점 | 빈틈(우리의 기회) |
|---|---|---|---|
| **자리톡** (임대인↔임차인 관리) | 월세 수납/미납 자동 리마인드, 장부 | 미납 알림 루프 | 수익률·세무 리포트 약함 |
| **직방/다방** | 매물 노출 UX, 세련된 디자인 | 카드/타이포 완성도 | 임대인 "운영/세무" 관점 부재 |
| **토스(금융 UX 기준)** | 자간·숫자 정렬·정보 위계 | 디자인 토큰/타이포 기준 | (직접 경쟁 아님) |
| 엑셀/수기 | 자유도 | — | 자동 집계·내보내기로 대체 |

**결론**: 경쟁앱은 "수납·알림"까지는 하지만 **① 운영 인사이트(수익률·현금흐름) ②
세무용 연간 리포트 ③ 이번 달 할 일(행동 루프)** 를 한 곳에 묶은 곳이 없다. 여기를 돌파구로 잡는다.

## 3. V2 결정 사항 (탑재 → 개발 완료)

### A. 디자인 시스템 개편 (디자이너 인사이트)
- **Pretendard** 가변 서체 적용, 자간 -0.011em(본문)/-0.022em(제목).
- 금액 `tabular-nums` 정렬, 카드/버튼/인풋 토큰화(`.rf-card`, `.rf-btn-primary`, `.rf-input`).
- 브랜드 마크(하우스 로고) + 모바일 앱바 정돈.

### B. 돌파구 기능 — 「인사이트」 페이지 (나머지 4개 인사이트 통합)
1. **현금흐름 KPI**: 월 예상수입 · 연 환산수입 · 점유율 · 미납 합계 (임대인)
2. **이번 달 할 일(Action Center)**: 만료 임박·미납·공실을 카드로 모아 해당 페이지로 딥링크 (중개인·마케터)
3. **매물별 수익률**: 보증금 대비 연 임대수입 비율, 공실 표시 (임대인)
4. **세무용 연간 임대소득 리포트**: 연도 선택 → 계약 기준 월별 임대수입 추정,
   총액/보증금 합계/월평균, **엑셀 내보내기**, 간주임대료 안내 (세무) ← *시장 빈틈 직격*
